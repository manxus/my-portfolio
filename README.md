# Build Verified — portfolio

React + Vite SPA with a game-style shell, Steam data, and two ways to edit content:

1. **Local dev (triple-click copyright on main menu)** — fast edits via the Vite admin API; see `.env.example` (`ADMIN_USER` / `ADMIN_PASS`).
2. **Decap CMS on the deployed site** — open **`/admin`** after configuring GitHub OAuth and `public/admin/config.yml` (repo + domain placeholders).

## Decap CMS (Git-backed, production)

- **URL:** `https://<your-domain>/admin`
- **Before first deploy:** Edit [public/admin/config.yml](public/admin/config.yml): set `repo`, `branch`, and `base_url` (no path; must match your live site origin, e.g. `https://my-app.vercel.app`).
- **GitHub:** Create an OAuth App ([GitHub docs](https://docs.github.com/en/apps/oauth-apps/building-oauth-apps/creating-an-oauth-app)).  
  - **Authorization callback URL:** `https://<your-domain>/api/complete` (must match `COMPLETE_URL` in Vercel).
- **Vercel env vars** (see [.env.example](.env.example)): `ORIGIN`, `COMPLETE_URL`, `ADMIN_PANEL_URL`, `OAUTH_CLIENT_ID`, `OAUTH_CLIENT_SECRET`.  
  `ORIGIN` may be a comma-separated list if you use preview URLs (e.g. prod + `*.vercel.app`).
- **Flow:** Log in with GitHub (account needs push access to the repo) → edit JSON → **Publish** creates a commit → Vercel rebuilds → site updates after deploy.
- **Not in CMS:** `src/data/steam-library.json` (generated by script), `steam-tierlist.json`, `steam-overrides.json` — keep editing those locally or extend `config.yml`.

Serverless OAuth handlers: [api/begin.cjs](api/begin.cjs) and [api/complete.cjs](api/complete.cjs) via [netlify-cms-oauth-provider-node](https://github.com/bericp1/netlify-cms-oauth-provider-node).

## React + Vite (template notes)

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react) uses [Babel](https://babeljs.io/) (or [oxc](https://oxc.rs) when used in [rolldown-vite](https://vite.dev/guide/rolldown)) for Fast Refresh
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react-swc) uses [SWC](https://swc.rs/) for Fast Refresh

## React Compiler

The React Compiler is not enabled on this template because of its impact on dev & build performances. To add it, see [this documentation](https://react.dev/learn/react-compiler/installation).

## Expanding the ESLint configuration

If you are developing a production application, we recommend using TypeScript with type-aware lint rules enabled. Check out the [TS template](https://github.com/vitejs/vite/tree/main/packages/create-vite/template-react-ts) for information on how to integrate TypeScript and [`typescript-eslint`](https://typescript-eslint.io) in the ESLint configuration.
